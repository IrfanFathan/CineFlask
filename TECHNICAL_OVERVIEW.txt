================================================================================
API ENDPOINTS
================================================================================

Authentication
POST   /auth/register          - Create new user account
POST   /auth/login             - Login and receive JWT token

Profiles
GET    /profiles               - List all profiles for current user
POST   /profiles               - Create new profile (max 4 per user)

Movies
GET    /movies                 - List all movies (supports ?q=search&profile_id=X)
GET    /movies/:id             - Get movie details + watchlist status
DELETE /movies/:id             - Delete movie and associated files

Upload
POST   /upload/init            - Initialize chunked upload session
POST   /upload/chunk           - Upload single chunk
POST   /upload/complete        - Finalize upload + fetch OMDB metadata
POST   /upload/cancel          - Cancel upload and cleanup chunks

Streaming
GET    /stream/:id             - Stream video with range request support

Subtitles
GET    /subtitles/:movieId     - List all subtitles for movie
POST   /subtitles/upload/:id   - Upload subtitle file (SRT/VTT)
DELETE /subtitles/:id          - Delete subtitle

Progress
GET    /progress/:movieId      - Get playback progress for profile
POST   /progress               - Save playback progress
GET    /progress               - Get all in-progress movies for profile

Watchlist
GET    /watchlist              - Get profile's watchlist
POST   /watchlist              - Add movie to watchlist
DELETE /watchlist/:movieId     - Remove from watchlist

================================================================================
INSTALLATION & DEPLOYMENT
================================================================================

Prerequisites:
- Node.js 16+ 
- npm or yarn
- 10GB+ free disk space (for video storage)

Quick Start:
1. Clone repository
   git clone <repo_url>
   cd CineFlask

2. Install dependencies
   npm install

3. Create .env file
   cp .env.example .env
   nano .env  # Add your OMDB_API_KEY and JWT_SECRET

4. Start server
   npm start

5. Access application
   http://localhost:3000

Production Deployment:

Option 1: PM2 (Recommended)
npm install -g pm2
pm2 start server.js --name cineflask
pm2 startup  # Auto-start on reboot
pm2 save

Option 2: Docker
docker build -t cineflask .
docker run -d -p 3000:3000 \
  -v $(pwd)/movies:/app/movies \
  -v $(pwd)/subtitles:/app/subtitles \
  -v $(pwd)/database.db:/app/database.db \
  -e OMDB_API_KEY=your_key \
  -e JWT_SECRET=your_secret \
  cineflask

Option 3: Systemd Service
sudo nano /etc/systemd/system/cineflask.service

[Unit]
Description=CineFlask Streaming Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/cineflask
ExecStart=/usr/bin/node server.js
Restart=on-failure
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target

sudo systemctl enable cineflask
sudo systemctl start cineflask

Reverse Proxy (Nginx):
server {
    listen 80;
    server_name movies.yourdomain.com;

    client_max_body_size 50G;  # Allow large file uploads

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }
}

SSL with Let's Encrypt:
sudo certbot --nginx -d movies.yourdomain.com

================================================================================
TROUBLESHOOTING
================================================================================

Issue: Upload fails at 100%
Solution: Check disk space and file permissions
  df -h
  chmod 755 movies/ uploads/

Issue: Video won't play
Solution: 
  1. Check browser console for errors
  2. Verify video codec compatibility
  3. Test with: ffprobe movies/filename.mp4

Issue: Subtitles not appearing
Solution:
  1. Verify .srt/.vtt file encoding (must be UTF-8)
  2. Check CORS headers for subtitle files
  3. Inspect browser Network tab for 404s

Issue: JWT token expired
Solution: Users need to re-login. Adjust token expiry in auth.js:
  expiresIn: '7d'  // Change from 24h to 7 days

Issue: Database locked
Solution: SQLite doesn't handle concurrent writes well
  1. Ensure only one server instance is running
  2. Consider migrating to PostgreSQL for high concurrency
